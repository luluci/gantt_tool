# ガントチャートツール

## 概要

- html/javascriptで構築するガントチャートツール
- 入力データはjsファイルで保存する
    - 入力データをjsonに変換したうえでjsファイルに保存する
    - ガントチャートツールhtmlからはscriptでjsファイルを読み込む
        - ファイルがない場合は空データから始める


## 詳細

- ガントチャート入力情報定義
    - 最上位要素としてdomainを設定する
        - domainは複数設定可能
    - domainの子要素としてprojectを設定する
        - projectは複数設定可能
    - projectの子要素としてversionを設定する
        - versionは複数設定可能
    - versionの子要素としてtaskを設定する
        - taskは複数設定可能
    - taskの子要素としてplanを設定する
        - planは複数設定可能
    - planが末端要素となり、複数の属性を設定する
        - 予定開始日、予定終了日属性
            - 開始日は手動で設定する
            - 終了日は設定された工数から自動算出する
        - 実績開始日、実績終了日属性
            - 開始日、終了日は手動で設定する
        - 稼働率属性
            - 稼働率=1.0=8hr/1day として設定する
            - ガントチャートでの表示と予定終了日に反映する
        - phase属性
            - ガントチャートツール内全体で共通のphase定義を持つ
            - phaseは複数設定可能とする
            - phaseは共通定義として追加削除可能とする
                - phaseはデフォルトで"all"を持つ。allは削除不可とする
                - phaseを追加した場合、全planに反映する。削除も同様
            - 各phaseに予定工数と実績工数を入力可能とする
                - allに入力した工数をガントチャートの表示に反映する
                - all以外にも工数を入力可能とするが、管理用に記録しておくための情報となる
        - case属性
            - ガントチャートツール内全体で共通のcase定義を持つ
            - caseはplanに対して1つのみ設定可能とする
            - ガントチャート表示において指定したcaseに該当するplanのみを表示するフィルタリングを可能とする
                - フィルタリングでは複数caseを指定可能とする
        - owner属性
            - ガントチャートツール内全体で共通のメンバー定義を持つ
            - メンバーから1人をplanにownerとして設定可能とする
        - case+ownerの組み合わせはtask内でユニークとする
        - 表示色属性
            - 予定工数のガントチャート上での表示色を設定可能とする
            - メンバー定義にデフォルト色を指定可能とし、デフォルトではその色を選択する
            - 表示色を変更したとき、case+ownerが同じ他のplanに対して同じ色を適用する
        - 休日属性
            - メンバーに対して個別に任意の休日を設定可能とする。複数設定可能とする
        - status属性
            - ガントチャートツール内全体で共通のstatus定義を持つ
            - statusはplanに対して1つのみ設定可能とする
- ツールGUI定義
    - 画面を上下で2ペインに分ける
        - 上部にはツール内共通データを定義するGUIを配置する
            - 「ガントチャートツール」というタイトルは表示しない
            - 以下項目を上ペインに配置する
                - データ保存欄
                - 入力ペイン表示切り替えボタン
                - phase入力欄、phaseフィルタ欄
                    - リストボックスで表示して一定の高さまでの表示とする
                    - 入力削除のインターフェースを備える
                    - 複数選択可能とする
                    - 選択解除ボタンを備える
                - case入力欄、caseフィルタ欄
                    - リストボックスで表示して一定の高さまでの表示とする
                    - 入力削除のインターフェースを備える
                    - 複数選択可能とする
                    - 選択解除ボタンを備える
                - member入力欄
                - status入力欄
                    - リストボックスで表示して一定の高さまでの表示とする
                    - 入力削除のインターフェースを備える
                - 共通休日入力欄
                    - リストボックスで表示して一定の高さまでの表示とする
                    - 入力削除のインターフェースを備える
        - 下部は左右で2ペインに分ける
            - 左ペインをdomain以下の情報を入力する入力ペインとする
            - 右ペインをガントチャート表示ペインとする
            - 左右ペインにそれぞれ垂直スクロールバーが表示され、個別にスクロールできるようにする
            - 水平スクロールバーは常時画面内の見える位置に表示する
            - 入力ペインの幅はドラッグで変更可能とする
            - 入力ペインは表示非表示を切り替え可能とする
                - 切り替えボタンは上部ペインに配置する
    - ガントチャート入力情報をガントチャート表示ペインに表示する
        - 年, 月, 日, 曜日をそれぞれ行を分けて表示する
            - 土曜,日曜は休日とする
        - 予定工数はセルの背景色を変えることで表示する
            - セル内にパディング(色のついていない箇所)ができないようにする
            - 何も入力されていないセルをシングルクリックしたとき、該当する日に対する工数入力操作とする
                - 1日あたり 8hr*稼働率 で工数をカウントする
                - セルをクリックして工数入力セルが増えた時、該当planにおけるphaseの"all"に加算する
            - 工数入力ありセルに対してクリックしたとき、稼働率を設定可能とする
                - planに設定したいる稼働率と同じ設定値の場合、セルには何も表示しない
                - planに設定したいる稼働率と異なる設定値の場合、セル内に値を表示する
            - 工数入力ありセルに対するドラッグの動作を下記のとおりとする
                - 連続した日付にデータが入力されていれば一塊のデータ入力とみなす
                    - 休日を挟んだ場合も一塊とする
                - 一塊のデータに対して、左右端の罫線をドラッグしたとき、ドラッグした分だけ工数を増減可能とする
                    - 左端を左側にドラッグしたとき、または、右端を右側にドラッグしたとき、該当する日を工数入力ありとする
                    - 左端を右側にドラッグしたとき、または、右端を左側にドラッグしたとき、該当する日からの工数入力削除とする
                - 一塊のデータに対して、左右端以外の部分をドラッグしたとき、入力済み工数を移動可能とする
            - 1列=1日あたりの工数は必ず 8hr*稼働率 となる
                - planに対して入力ありセルを記録するとき、記録する情報は日時と稼働率になる
                    - 稼働率はplanに設定した共通稼働率と同じ値なら省略する
        - 実績工数は細いバーを表示する
            - 実績工数バーはすべて同じ色とする
        - 全メンバーに共通の休日を設定可能とする
        - ガントチャート内でもdomain,project,version,taskは階層的に1行使って表示する
            - domain,project,versionの行は入力がないことがわかるように日付列は暗い色にする
            - domain,project,versionは2列から構築する仕様とする
                - day表示,domain,project,versionまではcolspan=2になるように設定する
                - task,planはそれぞれ1列ずつ使って配置する
                - taskはrowspan=<子要素のplanの数> となる
    - 入力ペイン
        - domain,project,version,task,planが階層的に常時表示されている仕様とする
            - plan入力情報も常時入力欄が表示されている形とする
            - 開始日、終了日、稼働率、工数入力、ケース、owner、表示色、statusを横並びにして高さを可能な限り小さくする
            - 開始日、終了日、工数入力の予定と実績があるデータは縦並びにする
        - domain,project,version,task,planの名前入力欄
            - クリックしてフォーカルを当てるとテキストボックスに切り替わる仕様とする
                - enter押下などによりフォーカルが外れるとテキスト要素に戻って入力内容を確定する
        - フォーカスが当たっているplanについて、ガントチャート表示ペインでも選択していることがわかるように強調表示をする
- 入力データ保存
    - Ctrl+Sで入力データの保存処理を実行する
        - htmlと同じフォルダにgantt_data.jsとして保存する
            - htmlは起動時に同じフォルダ内にあるgantt_data.jsを参照する
        - データ保存時はポップアップを表示する
        - 入力データに変更があったとき、保存せずに画面の更新やクローズをするときは警告を表示する
        - wri.exe以外でhtmlを開いているときは FS access api を使用して保存する
        - wri.exeでhtmlを開いているときはwri APIを使用する
            - wri.exeで開いているときは UserAgent に wri/1.0 が追加されるので、これを使って判定する
            - wri API
                - ファイル保存API
```js
wri.io.file.SaveTo(FilePath, text);
```
- localize
    - Date.prototype.toISOString()
        - タイムゾーンのケアが必要
```js
function toDateString(date) {
    const diff = date.getTimezoneOffset() * 60 * 1000;
    date = new Date(date - diff);
    return date.toISOString().slice(0, 10);
}
```
